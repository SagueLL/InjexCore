name: compare-frameworks
description: Compare frameworks, libraries, platforms, or tools across architecture, ecosystem, DX, scalability, deployment, maintainability, and use-case fit, producing a tradeoff-aware recommendation. Use when the user asks to compare, evaluate, or pick between two or more technologies (e.g. "FastAPI vs Flask", "PyTorch vs TensorFlow", "Postgres vs Mongo for X").
---

# PURPOSE

Compare frameworks, libraries, platforms, tools, or technologies
in a structured, technically rigorous, and context-aware manner.

The skill should identify tradeoffs, strengths, weaknesses,
and suitability for specific use cases.

---

# RESPONSIBILITIES

- Compare architecture and design philosophy
- Compare developer experience
- Compare ecosystem maturity
- Compare scalability characteristics
- Compare deployment complexity
- Compare maintainability
- Compare ML/AI suitability when relevant
- Compare performance characteristics
- Compare documentation quality
- Identify tradeoffs and risks
- Recommend best-fit scenarios

---

# NON-GOALS

- Do not produce hype-based recommendations
- Do not recommend based solely on popularity
- Do not ignore tradeoffs
- Do not oversimplify technical differences

---

# TOOLS

- WebSearch
- WebFetch
- Read

---

# INPUTS

- Framework or technology names
- Context and use case
- Constraints and priorities
- Evaluation criteria

---

# WORKFLOW

1. Understand the target use case
2. Research each framework/tool
3. Compare across relevant dimensions
4. Identify tradeoffs and limitations
5. Analyze ecosystem maturity and long-term viability
6. Produce contextual recommendations

---

# OUTPUT FORMAT

# Context
# Evaluation Criteria
# Framework A Analysis
# Framework B Analysis
# Tradeoffs
# Best Fit Scenarios
# Risks and Limitations
# Recommendation
# Confidence Level

---

# QUALITY BAR

- Be context-aware
- Explicitly discuss tradeoffs
- Separate facts from opinions
- Avoid absolutist conclusions
- Prioritize practical engineering considerations

---

# FAILURE BEHAVIOR

If reliable information is insufficient:
- explicitly state uncertainty
- avoid unsupported comparisons
- identify missing information
