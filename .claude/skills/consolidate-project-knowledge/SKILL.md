# PURPOSE

Consolidate fragmented project knowledge into a coherent and unified understanding.

The skill should reduce information fragmentation and improve project visibility.

---

# RESPONSIBILITIES

- Consolidate project knowledge
- Consolidate architecture information
- Consolidate roadmap information
- Consolidate memory information
- Consolidate agent ecosystem knowledge
- Consolidate technical decisions
- Detect overlapping information
- Detect conflicting information
- Improve project visibility

---

# NON-GOALS

- Do not overwrite source documents
- Do not invent project knowledge
- Do not remove information without evidence
- Do not make architectural decisions

---

# TOOLS

- Read
- Glob
- Grep

---

# INPUTS

- Documentation
- Memory files
- Roadmaps
- Architecture documents
- Agent definitions
- Skills definitions

---

# WORKFLOW

1. Gather project information sources
2. Identify overlapping information
3. Identify conflicting information
4. Consolidate relevant knowledge
5. Build unified project understanding
6. Generate consolidated knowledge report

# QUALITY BAR

- Preserve factual accuracy
- Reduce fragmentation
- Maintain consistency
- Preserve traceability
- Focus on actionable consolidation

---

# FAILURE BEHAVIOR

If project knowledge is incomplete:

- explicitly state limitations
- consolidate only observable information
- identify missing sources
