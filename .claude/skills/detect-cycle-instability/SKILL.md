name: detect-cycle-instability
description: Detect instability, abnormal variability, and inconsistent behavior within machine operational cycles — unstable cycle durations, irregular timing, fluctuating process metrics — flagging operational reliability risks without assuming imminent failure. Use when the user asks to evaluate cycle stability, cycle-time variance, process consistency, or operational reliability across production cycles.
---

# PURPOSE

Detect instability, inconsistency, and abnormal variability
within machine operational cycles.

The skill should identify operational reliability risks
and unstable process behavior.

---

# RESPONSIBILITIES

- Detect unstable cycle durations
- Detect inconsistent operational behavior
- Detect abnormal cycle transitions
- Detect operational fluctuations
- Detect irregular process timing
- Evaluate operational reliability
- Support predictive maintenance workflows

---

# NON-GOALS

- Do not assume instability implies imminent failure
- Do not fabricate operational explanations
- Do not oversimplify process variability
- Do not ignore contextual uncertainty

---

# TOOLS

- Read
- Write

---

# INPUTS

- Cycle telemetry
- Operational signals
- Stability thresholds (optional)

---

# WORKFLOW

1. Analyze cycle variability
2. Detect instability patterns
3. Analyze operational fluctuations
4. Evaluate process consistency
5. Identify high-risk instability signals
6. Produce structured instability report

---

# OUTPUT FORMAT

# Instability Overview
# Unstable Cycles
# Operational Variability
# Timing Inconsistencies
# Reliability Risks
# Recommendations
# Confidence Level

---

# QUALITY BAR

- Preserve operational context
- Prioritize meaningful instability analysis
- Avoid simplistic stability assumptions
- Focus on operational relevance
- Preserve interpretability

---

# FAILURE BEHAVIOR

If operational visibility is incomplete:
- explicitly state limitations
- avoid unsupported instability conclusions
- analyze only observable evidence
