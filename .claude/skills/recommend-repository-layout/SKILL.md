# PURPOSE

Recommend repository structures that improve scalability, maintainability, discoverability, and long-term project health.

The skill should balance simplicity with future growth requirements.

---

# RESPONSIBILITIES

- Recommend repository layouts
- Recommend folder structures
- Recommend module boundaries
- Recommend documentation organization
- Recommend data organization
- Recommend configuration organization
- Improve scalability
- Improve maintainability

---

# NON-GOALS

- Do not enforce restructuring
- Do not recommend unnecessary complexity
- Do not ignore project context
- Do not optimize for aesthetics alone

---

# TOOLS

- Read
- Glob
- Grep

---

# INPUTS

- Current repository structure
- Project goals
- Team size
- Repository growth expectations
- Technical constraints

---

# WORKFLOW

1. Analyze repository architecture
2. Assess growth trajectory
3. Identify structural weaknesses
4. Evaluate organizational alternatives
5. Recommend improved layouts
6. Produce architecture recommendations

---

# OUTPUT FORMAT

# Current Structure Assessment
# Layout Recommendations
# Folder Organization
# Module Organization
# Documentation Strategy
# Migration Considerations
# Recommendations
# Confidence Level

---

# QUALITY BAR

- Prioritize maintainability
- Prioritize scalability
- Avoid unnecessary complexity
- Preserve project continuity
- Produce realistic recommendations

---

# FAILURE BEHAVIOR

If repository visibility is incomplete:

- explicitly state limitations
- recommend only evidence-supported improvements
- identify missing context
