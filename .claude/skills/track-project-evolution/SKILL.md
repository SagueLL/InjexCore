# PURPOSE

Track and explain project evolution over time.

The skill should identify meaningful changes, milestones, and patterns in project development.

---

# RESPONSIBILITIES

- Analyze project evolution
- Identify major milestones
- Identify architecture evolution
- Identify roadmap progression
- Identify ecosystem growth
- Identify repository evolution
- Identify significant decisions
- Preserve project history

---

# NON-GOALS

- Do not invent historical events
- Do not rewrite project history
- Do not exaggerate progress
- Do not generate unsupported timelines

---

# TOOLS

- Read
- Glob
- Grep

---

# INPUTS

- Previous versions
- Current project state
- Roadmaps
- Documentation
- Architecture files
- Repository artifacts

---

# WORKFLOW

1. Analyze project history
2. Compare historical and current state
3. Identify significant changes
4. Identify milestones and trends
5. Assess project evolution
6. Generate evolution report

---

# OUTPUT FORMAT

# Evolution Overview

# Major Milestones

# Architecture Evolution

# Ecosystem Evolution

# Documentation Evolution

# Repository Evolution

# Key Decisions

# Lessons Learned

# Confidence Level

---

# QUALITY BAR

- Preserve historical accuracy
- Focus on meaningful changes
- Avoid unnecessary detail
- Maintain contextual awareness
- Highlight significant evolution

---

# FAILURE BEHAVIOR

If historical visibility is incomplete:

- explicitly state limitations
- analyze only observable history
- identify missing references

---

# FUTURE EXTENSIONS

Possible future capabilities:

- Automated changelog generation
- Milestone tracking
- Project timeline visualization
- Growth trend analysis
